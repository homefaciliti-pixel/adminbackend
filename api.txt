2. USERS MANAGEMENT
---
Endpoint: GET /api/users
Description: Retrieve list of users. Supports search via query parameters.
Query Params (Optional):
- query: Searches name, email, or mobile (LIKE)
- name: Filter by name (LIKE)
- email: Filter by email (LIKE)
- mobile: Filter by mobile (LIKE)
Response Data: Array of Users

Endpoint: GET /api/users/:id
Description: Fetch details of a single user by ID.
Response Data: Single User Object

Endpoint: POST /api/users
Request Body:
{
  "name": "John Doe",
  "email": "john@example.com",
  "mobile": "9876543210",
  "address": "123 Street Name, Delhi"
}

Endpoint: DELETE /api/users/:id
Description: Delete user by ID.

3. CATEGORIES MANAGEMENT
---
Endpoint: GET /api/categories
Description: Retrieve list of categories. Supports searching/filtering via query parameters.
Query Params (Optional):
- title OR categoryName: Search category name (LIKE)
- parent OR mainCategory: Filter by parent category (LIKE)
- status OR emailStatus: Filter by status (true or false / 1 or 0)
Response Data: Array of Categories

Endpoint: POST /api/categories
Description: Add a new category.
Request Body Option A (Standard):
{
  "title": "Kitchen Cleaning",
  "parent": "Cleaning",
  "image": "https://url-to-image.png",
  "status": true
}
Request Body Option B (Alias Fields):
{
  "categoryName": "Kitchen Cleaning",
  "mainCategory": "Cleaning",
  "image": "https://url-to-image.png",
  "emailStatus": true
}

Endpoint: PUT /api/categories/:id
Description: Update category details. Accepts any properties from Option A or Option B.

Endpoint: DELETE /api/categories/:id
Description: Delete category by ID.

6. PARTNERS (VENDORS)
---
Endpoint: GET /api/partners
Description: Retrieve list of partners. Supports filtering and searching via query parameters.
Query Params (Optional):
- name: Search by partner name (LIKE)
- mobile: Search by mobile number (LIKE)
- city: Filter by city (LIKE)
- state: Filter by state (LIKE)
- date: Filter by createdAt date (LIKE)
- status: Filter by status (1 or 0 / true or false)
- isApproved: Filter by approval status (1 or 0 / true or false)
Response Data: Array of Partners with KYC data

Endpoint: GET /api/partners/pending
Description: Retrieve list of partners pending approval (isApproved = 0). Supports the same query parameters as the GET /api/partners route.

Endpoint: GET /api/partners/:id
Description: Get full details of a single partner by ID.

Endpoint: POST /api/partners
Request Body:
{
  "name": "Suresh Kumar",
  "email": "suresh@gmail.com",
  "mobile": "9911223344",
  "city": "Delhi",
  "state": "Delhi",
  "locality": "Dwarka",
  "address": "Sector 12, Dwarka, Delhi",
  "image": "https://url.png",
  "gender": "Male",
  "experience": "4 Years",
  "services": ["Plumbing", "AC Repair"],
  "aadhaarNumber": "123456789012",
  "panNumber": "ABCDE1234F",
  "bankName": "ICICI Bank",
  "accountNumber": "98765432101",
  "ifscCode": "ICIC0000456",
  "documents": ["aadhaar.pdf"]

Endpoint: PUT /api/partners/:id
Description: Update/Edit partner details.

Endpoint: PUT /api/partners/:id/disapprove

Description: Delete partner by ID.

7. EARNINGS
---
Endpoint: GET /api/earnings/bookings
Description: Fetch booking transactions. Supports filters and search, and returns aggregates.
Query Params (Optional):
- query: Searches transactionId, paymentMethod, extraServicePaymentMethod, orderDate
- transactionId: Filter by transactionId (LIKE)
- paymentMethod: Filter by paymentMethod (LIKE)
- orderDate: Filter by orderDate (LIKE)
Response Schema:
{
  "success": true,
  "totalBookingEarnings": 2800.00,
  "totalTransactions": 3,
  "filteredBookingEarnings": 2800.00,
  "filteredTransactions": 3,
  "data": [ ... ]
}

Endpoint: POST /api/earnings/bookings
Request Body:
{
  "transactionId": "TXN1004",
  "serviceAmount": 500.00,
  "paymentMethod": "UPI",
  "extraServiceAmount": 50.00,
  "extraServicePaymentMethod": "Cash",
  "totalAmount": 550.00,
  "orderDate": "15-05-2026"
}

Endpoint: GET /api/earnings/subscriptions
Description: Fetch subscription purchases. Supports filters and search, and returns aggregates.
Query Params (Optional):
- query: Searches partnerName, paymentMethod, status, purchaseDate
- partnerName: Filter by partner name (LIKE)
- paymentMethod: Filter by payment method (LIKE)
- status: Filter by payment status (Paid, etc.)
- purchaseDate: Filter by purchase date (LIKE)
Response Schema:
{
  "success": true,
  "totalSubscriptionsEarnings": 200.00,
  "totalPlans": 2,
  "filteredSubscriptionsEarnings": 200.00,
  "filteredPlans": 2,
  "data": [ ... ]
}

Endpoint: POST /api/earnings/subscriptions
Request Body:
{
  "partnerName": "Govind",
  "amount": 100.00,
  "paymentMethod": "UPI",
  "purchaseDate": "16-05-2026",
  "status": "Paid"
}

9. CONFIGURATION & SETTINGS
---
Endpoints:
- Banners: GET|POST|PUT|DELETE /api/settings/banners
  Description: Manage banners. GET supports query filter parameter "title" (LIKE).
  Body sample: { "title": "Offer", "image": "url", "status": true }

- States: GET|POST|PUT|DELETE /api/settings/states
  Description: Manage states. GET supports query filter parameter "name" (LIKE).
  Body sample: { "name": "Haryana", "status": true }

- Cities: GET|POST|PUT|DELETE /api/settings/cities
  Description: Manage cities. GET supports query filter parameters "cityName" (LIKE), "stateName" (LIKE).
  Body sample: { "cityName": "Gurugram", "stateName": "Haryana", "status": true }

- Localities: GET|POST|PUT|DELETE /api/settings/localities
  Description: Manage localities. GET supports query filter parameters "localityName" (LIKE), "cityName" (LIKE), "stateName" (LIKE).
  Body sample: { "localityName": "Sector 54", "cityName": "Gurugram", "stateName": "Haryana", "status": true }

- Commission Setting: GET|PUT /api/settings/commission
  Description: Manage commission settings.
  GET returns: { "success": true, "commissionRate": 10.0 }
  PUT Request Body: { "commissionRate": 15.0 }

- Reviews & Ratings: GET|POST|PUT|DELETE /api/settings/reviews
  Body sample: { "userName": "Rahul", "partnerName": "Govind", "rating": 5.0, "reviewText": "Good" }

- Notifications: GET|POST|PUT|DELETE /api/settings/notifications
  Description: Manage notification logs. GET supports query filter parameters "title" (LIKE), "audience" (LIKE), "status" (1 or 0).
  Body sample: { "title": "Alert", "message": "KYC needed", "audience": "All Partners" }
  Body sample for sending: { "isSent": true }

10. REPORTS
---
Endpoints:
- Users Report: GET /api/reports/users
- Partners Report: GET /api/reports/partners
- Earnings Report: GET /api/reports/earnings
- Subscriptions Report: GET /api/reports/subscriptions

Query Params (Optional):
- startDate: Filter start date (YYYY-MM-DD or DD-MM-YYYY)
- endDate: Filter end date (YYYY-MM-DD or DD-MM-YYYY)
- query: General search keyword (LIKE)
- export: Pass "csv" to download report as CSV file

11. SUPPORT TICKETS
---
Endpoints:
- Support Ticket List: GET /api/support
  Query Params (Optional):
  - query: Search UserName, Email, Subject, Message (LIKE)
  - status: Filter by status (Open, Closed, etc.)
- Ticket Details: GET /api/support/:id
- Create Ticket: POST /api/support
  Request Body: { "userName": "Rahul", "email": "rahul@gmail.com", "mobile": "9876543210", "subject": "AC Issue", "message": "AC not cooling" }
- Update Ticket: PUT /api/support/:id
  Request Body: { "status": "Closed" }
- Delete Ticket: DELETE /api/support/:id
