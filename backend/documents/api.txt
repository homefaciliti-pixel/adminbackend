2. USERS MANAGEMENT
----------------------------------------------------------------------
Endpoint: GET /api/users
Description: Retrieve a list of users.
Query Parameters (Optional):
  - query: Search by name, email, or mobile (LIKE match)
  - name: Filter by name (LIKE match)
  - email: Filter by email (LIKE match)
  - mobile: Filter by mobile number (LIKE match)
Response Data: Array of User objects.

Endpoint: GET /api/users/:id
Description: Fetch details of a single user by ID.
Response Data: Single User object.

Endpoint: POST /api/users
Description: Create a new user.
Request Body (JSON):
{
  "name": "John Doe",
  "email": "john@example.com",
  "mobile": "9876543210",
  "address": "123 Street Name, Delhi"
}
Response Data: Created User object.

Endpoint: DELETE /api/users/:id
Description: Delete a user by ID.
Response Data: { "success": true, "message": "User deleted successfully" }

3. CATEGORIES MANAGEMENT
----------------------------------------------------------------------
Endpoint: GET /api/categories
Description: Retrieve a list of categories.
Query Parameters (Optional):
  - title OR categoryName: Filter by category name (LIKE match)
  - parent OR mainCategory: Filter by parent category name (LIKE match)
  - status OR emailStatus: Filter by status ('true'/'false' or '1'/'0')
Response Data: Array of Category objects. Note: parent category 'None' maps to database NULL.

Endpoint: POST /api/categories
Description: Add a new category.
Request Body Option A (Standard):
{
  "title": "Kitchen Cleaning",
  "parent": "Cleaning",        // Use 'None' if it has no parent
  "image": "https://url-to-image.png",
  "status": true
}
Request Body Option B (Alias Fields):
{
  "categoryName": "Kitchen Cleaning",
  "mainCategory": "Cleaning",  // Use 'None' if it has no parent
  "image": "https://url-to-image.png",
  "emailStatus": true
}
Response Data: Created Category object.

Endpoint: PUT /api/categories/:id
Description: Update category details. Accepts any properties from Option A or Option B.
Response Data: Updated Category object.

Endpoint: DELETE /api/categories/:id
Description: Delete category by ID. Cascade-deletes all sub-categories having this category title as parent.
Response Data: { "success": true, "message": "Category and its sub-categories deleted successfully" }

Response Data: Array of Service objects. If the table contains `category_id`, it is mapped to both `category_id` and `categoryId`.

Endpoint: PUT /api/services/:id
Description: Update service details.
Request Body (JSON): Accepts title, price, image, description, status, categoryId.

Endpoint: DELETE /api/services/:id

5. ORDERS (SERVICE REQUESTS)
----------------------------------------------------------------------
Endpoint: GET /api/orders
Description: Retrieve a list of all service requests/orders.
Response Data: Array of Order objects. Assigned vendor name 'None' or no vendor maps to database NULL (returns '-' in response).

Endpoint: POST /api/orders
Description: Create a new order.
Request Body (JSON):
{
  "serviceRequestNumber": "REQ2001",
  "serviceName": "AC Repair",
  "serviceAmount": 499.00,
  "slotTime": "11:00 AM - 01:00 PM",
  "serviceDate": "20-05-2026",
  "city": "Delhi",
  "locality": "Connaught Place",
  "status": "Pending",
  "vendorName": "-",          // Use '-' or null if not yet assigned
  "address": "Block C, Connaught Place, Delhi"
}
Response Data: Created Order object.

Endpoint: PUT /api/orders/:id
Description: Update order details (e.g. assign vendor, update status, change slots). Resolves vendorName by phone number using 'vendorPhone', 'phone', or 'mobile' in request body if provided. Returns 404 if phone number does not map to any partner.
Request Body (JSON): Supports status, vendorName, vendorPhone, phone, mobile, slotTime, serviceDate, city, locality, address.
Response Data: Updated Order object.

Endpoint: PUT /api/orders/:id/assign
Description: Assign or unassign order to/from partner. Resolves vendor name using phone number lookup (via 'vendorPhone', 'phone', or 'mobile' parameters) in the database. Updates status to 'Assigned' if resolved, or 'Pending' if unassigned.
Request Body (JSON): Supports vendorName, vendorPhone, phone, mobile.
Response Data: { "success": true, "message": "Order assigned successfully", "data": Order }

Endpoint: DELETE /api/orders/:id
Description: Delete an order by ID.
Response Data: { "success": true, "message": "Order deleted successfully" }

9. CONFIGURATION & SETTINGS
----------------------------------------------------------------------
- BANNERS:
  * GET /api/settings/banners: List all banners. Query param "title" filters (LIKE match).
  * POST /api/settings/banners: Add banner. Body: { "title": "Offer", "image": "url", "status": true }
  * PUT /api/settings/banners/:id: Update banner.
  * DELETE /api/settings/banners/:id: Delete banner.

- STATES:
  * GET /api/settings/states: List states. Query param "name" filters (LIKE match).
  * POST /api/settings/states: Add state. Body: { "name": "Haryana", "status": true }
  * PUT /api/settings/states/:id: Update state.
  * DELETE /api/settings/states/:id: Delete state. Cascade-deletes all cities and localities under this state name.

- CITIES:
  * GET /api/settings/cities: List cities. Query params "cityName" and "stateName" filter.
  * POST /api/settings/cities: Add city. Body: { "cityName": "Gurugram", "stateName": "Haryana", "status": true }
  * PUT /api/settings/cities/:id: Update city.
  * DELETE /api/settings/cities/:id: Delete city. Cascade-deletes all localities under this city name.

- LOCALITIES:
  * GET /api/settings/localities: List localities. Query params "localityName", "cityName", "stateName" filter.
  * POST /api/settings/localities: Add locality. Body: { "localityName": "Sector 54", "cityName": "Gurugram", "stateName": "Haryana", "status": true }
  * PUT /api/settings/localities/:id: Update locality.
  * DELETE /api/settings/localities/:id: Delete locality.

- COMMISSION SETTING:
  * GET /api/settings/commission: Retrieve commission rate. Response: { "success": true, "commissionRate": 10.0 }
  * PUT /api/settings/commission: Update commission rate. Body: { "commissionRate": 15.0 }

- REVIEWS & RATINGS:
  * GET /api/settings/reviews: Retrieve reviews list.
  * POST /api/settings/reviews: Add review. Body: { "userName": "Rahul", "partnerName": "Govind", "serviceName": "AC Repair", "rating": 5.0, "reviewText": "Good" }
  * PUT /api/settings/reviews/:id: Toggle review status. Body: { "status": true }
  * DELETE /api/settings/reviews/:id: Delete review.

- NOTIFICATIONS LOGS:
  * GET /api/settings/notifications: List notification logs. Query params "title", "audience", "status" filter.
  * POST /api/settings/notifications: Add notification log. Body: { "title": "Alert", "message": "Details", "audience": "All Partners", "status": true }
  * PUT /api/settings/notifications/:id: Update log / Mark as sent. Body: { "isSent": true }
  * DELETE /api/settings/notifications/:id: Delete notification.

10. REPORTS
----------------------------------------------------------------------
Endpoints:
  - Users Report: GET /api/reports/users
  - Partners Report: GET /api/reports/partners
  - Booking Earnings Report: GET /api/reports/earnings
  - Subscriptions Report: GET /api/reports/subscriptions

Query Parameters (Required):
  - startDate: Start date filter (DD-MM-YYYY or YYYY-MM-DD format)
  - endDate: End date filter (DD-MM-YYYY or YYYY-MM-DD format)
  - query: General search keyword (LIKE match) — pass empty string "" to skip filtering
  - export: Pass "csv" to download report as a raw CSV file — pass empty string "" for JSON response

Error (400) if any parameter is missing:
{ "success": false, "message": "startDate, endDate, query, and export parameters are required" }

Example:
  GET /api/reports/users?startDate=01-05-2026&endDate=31-05-2026&query=Rahul&export=
  GET /api/reports/users?startDate=01-05-2026&endDate=31-05-2026&query=&export=csv

13.4 BOOKING WORK LIFECYCLE
---------------------------
* GET /api/bookings/:id (PROTECTED)
  - Fetches full details of a single booking by ID.
* POST /api/bookings/:id/start (PROTECTED)
  - Moves booking status to 'In Progress'.
* POST /api/bookings/send-complete-otp (PROTECTED)
  - Generates and sends a 4-digit completion OTP to the customer phone number. Body: { "phone": "...", "countryCode": "...", "orderId": "..." }
* POST /api/bookings/:id/complete (PROTECTED)
  - Completes booking, processes commission, updates wallet/earnings, and logs transaction. Requires OTP verification. Body: { "paymentMethod": "Cash" / "UPI", "otp": "...", "customerPhone": "..." }. For Cash, 25% commission is added to payToCompany (due to company) and wallet balance is not credited. For UPI, standard commission rate is used and partner's share is credited to wallet balance. Bypass OTP with '1234'.

13.5 DYNAMIC REGISTRATION DROPDOWNS
-----------------------------------
* GET /api/metadata/locations (PUBLIC)
  - Fetches nested active states, cities, and localities hierarchy.
* GET /api/metadata/categories (PUBLIC)
  - Fetches active categories and nested subcategories with service title lists.
* GET /api/countries (PUBLIC)
  - Returns a list of all countries with flags and international calling codes.

13.6 PHONE OTP VERIFICATION
---------------------------
* POST /api/auth/send-otp (PUBLIC)
  - Request a 4-digit OTP code for a phone number. Body: { "phone": "..." }. Returns: { "success": true, "message": "...", "otp": "..." } (returned for debugging/testing).
* POST /api/auth/verify-otp (PUBLIC)
  - Verify the requested OTP code for a phone number. Body: { "phone": "...", "otp": "..." }. Returns: { "success": true, "message": "..." }.
* POST /api/partner/location (and alias POST /api/partner/update-location) (PROTECTED)
  - Update partner coordinates and time. Body: { partnerId, token, lat, lon, time }.
* GET /api/partner/location (and aliases GET /api/partner/location/:id & GET /api/partner/:id/location) (PUBLIC / PROTECTED)
  - Retrieve partner coordinates and time.
* POST /api/partner/status (and alias PUT /api/partner/status) (PROTECTED)
  - Update partner status (online/offline) and coordinates. Body: { "status": true/false/1/0, "lat": "...", "lon": "...", "time": "..." }.
