1. PARTNER AUTHENTICATION
----------------------------------------------------------------------

1.1 REGISTER AS NEW PARTNER
---------------------------
Endpoint: POST /api/auth/register
Request Format: multipart/form-data
Description: Registers a new partner. Uploads documents and hashes the password.

Request Fields:
- Text Fields:
  * name (String, Required) - e.g., "Amit Kumar"
  * phone (String, Required) - e.g., "9911223344" (Saves to 'mobile' column in database)
  * email (String, Required) - e.g., "amit@gmail.com"
  * address (String, Required) - e.g., "Sector 15, Jaipur"
  * state (String, Required) - e.g., "Rajasthan"
  * city (String, Required) - e.g., "Jaipur"
  * locality (String, Required) - e.g., "Mansarovar"
  * password (String, Required) - e.g., "secure123"
  * gender (String, Optional) - "Male" or "Female"
  * category (String, Optional) - e.g., "Electrician"
  * subCategory (String, Optional) - e.g., "Repair"
  * hasVehicle (String, Optional) - "Yes" or "No"
  * services (String, Optional) - Comma-separated services, e.g., "AC Repair,Fan Repair"
  * aadharNumber (String, Optional) - e.g., "123456789012"
  * panNumber (String, Optional) - e.g., "ABCDE1234F"
  * bankName (String, Optional) - e.g., "State Bank of India"
  * accountHolder (String, Optional) - e.g., "Amit Kumar"
  * accountNumber (String, Optional) - e.g., "30291827361"
  * ifscCode (String, Optional) - e.g., "SBIN0001234"

- File Upload Fields (MANDATORY):
  * profileImage - File upload for partner's photo (saves to 'image' column)
  * aadharFront - File upload for Aadhar Card front
  * aadharBack - File upload for Aadhar Card back
  * panImage - File upload for PAN Card photo
  * policeVerification - File upload for Police Verification document (saves to 'policeVerificationImage' column)

Response Example (Status 201 Created):

1.2 LOGIN PARTNER
-----------------

Request Format: application/json
Description: Logs in a partner using their phone number and password.

Request Body:
{

  "password": "secure123"

7.5 COMPLETE BOOKING WORK & COMMISSIONS
---------------------------------------
Endpoint: POST /api/bookings/:id/complete (PROTECTED)
Description: Completes booking, processes commission, updates wallet/earnings, and logs transaction details inside database tables. Requires OTP verification.
Request Body (JSON):
{
  "paymentMethod": "Cash",   // Can be "Cash" or "UPI". Defaults to "UPI"
  "otp": "8676",             // Required. Verification OTP sent to customer phone. (Bypass: '1234')
  "customerPhone": "9876543210" // Optional. Defaults to '9876543210'.
}
Note on Commissions and Balances:
- If paymentMethod is "Cash", the commission rate is 25%. Since the partner collects the cash directly, no walletBalance is added. Instead, the 25% commission is added to "payToCompany" (which the partner owes to the company).
- If paymentMethod is "UPI", the standard commission rate (from settings, default 10%) is used. The partner's share is credited to their "walletBalance".
Response Example (Status 200 OK - Cash Payment):
{
  "success": true,
  "message": "Work completed successfully and earnings updated!",
  "status": "completed",
  "earnings": {
    "serviceAmount": 500.00,
    "paymentMethod": "Cash",
    "commissionRate": 25,
    "commissionAmount": 125.00,
    "partnerShare": 375.00,
    "walletBalanceAdded": 0.00,
    "payToCompanyAdded": 125.00
  }
}
